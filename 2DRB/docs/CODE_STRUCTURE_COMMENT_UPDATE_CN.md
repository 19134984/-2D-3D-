# 2DRB 单文件结构化注释整理说明

## 修改目的

这次修改的目标不是拆分文件，也不是重构求解流程，而是继续保持 `2DRB.F90` 为单文件形式，只通过统一的注释块把代码结构整理清楚，方便后续阅读、查找和维护。

本次修改遵循两个原则：

- 不改变主程序和各个子程序的调用逻辑
- 不改动数值算法、边界条件实现和后处理计算公式

## 主要修改内容

### 1. 为每个子程序增加统一格式的头部说明

在每个 `subroutine` 定义之前，统一增加了如下形式的注释块：

```fortran
!===================================================================================================
! 子程序: xxx
! 作用: ...
! 用途: ...
!===================================================================================================
```

其中：

- `子程序`：写明当前子程序名称
- `作用`：说明这个子程序主要负责什么计算或输出
- `用途`：说明这个子程序在主程序或其它后处理流程中是在哪里被调用的

这样在查看单文件时，不需要先读完整个实现，就能先知道每个子程序的职责。

### 2. 为每个子程序增加统一格式的结束说明

在每个 `end subroutine xxx` 后面，统一增加了结束说明：

```fortran
!===================================================================================================
! xxx 结束: ...
!===================================================================================================
```

这样可以在长文件中更清楚地看到每个子程序的结束位置，避免在阅读连续的多个子程序时出现边界不清的问题。

### 3. 保留单文件结构，不做多文件模块化

本次没有把代码拆成多个 `.f90` 文件，也没有引入额外接口层。仍然保持当前单文件组织方式，只通过注释把代码按功能区块和子程序边界梳理清楚。

这意味着：

- 服务器上的编译和运行方式不需要变化
- 原来的主程序调用顺序保持不变
- 后续如果继续修改参数、case 或边界条件，仍然可以在单文件内完成

## 涉及的子程序范围

本次注释整理覆盖了 `2DRB.F90` 中的全部主要子程序，包括但不限于：

- 初始化：`initial`
- 流场求解：`collision`、`streaming`、`bounceback`、`macro`
- 温度求解：`collisionT`、`streamingT`、`bouncebackT`、`macroT`
- 收敛检查：`check`
- 输出与备份：`output_binary`、`backupData`、`output_Tecplot`、`dumpstring`
- Nu/Re 后处理：`calNuRe`、`SideHeated...`、`RBcalc...`
- 流函数/涡量后处理：`calc_psi_vort_and_output`、`output_Tecplot_psi_vort`
- 样条插值辅助程序：`spline_natural`、`splint`

## 本次修改的直接效果

- 单文件中每个子程序的边界更清晰
- 能更快定位“某个功能在哪个子程序里”
- 后续继续维护单文件时，不容易把不同阶段的代码混在一起看
- 适合当前代码规模下的轻量整理方式

## 未改动部分

以下内容本次没有修改：

- 主程序时间推进逻辑
- 流场和温度场碰撞/迁移/边界处理公式
- Nu、Re、流函数、涡量等后处理算法
- 特征长度自动切换逻辑

## 编译检查说明

本次修改后做了最小编译检查。

结果表明：

- 本次新增的注释没有引入新的语法问题
- `2DRB.F90` 仍然保留原有编译问题：`SG = 1.0d0 - 0.5d0*s_j` 这一处把非常量写成了 `parameter`

也就是说，这次提交的主要变化是代码注释和结构可读性整理，不是数值逻辑修复。
